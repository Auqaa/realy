# Рязанский квест (offline QR-приключение)

Геймифицированный туризм с QR-квестами, офлайн-очередью сканов и лидербордом.

### Быстрый запуск (Python, без Node)

```bash
python local_server.py
```

Откройте `http://localhost:8000`. Карта: **2GIS MapGL**, пешеходные маршруты: **2GIS Routing API** (ключ по умолчанию в коде; можно задать `TWOGIS_KEY` в переменных окружения). При обновлении схемы БД старый `backend/data/db.json` переименовывается в `.bak.json` и создаётся новый файл.

Озвучка гида для кремля: положите файл в `site/media/kremlin-guide.mp3` (или скопируйте из своих загрузок).

## 1) Предварительные требования

1. **Node.js 18+** (LTS).
2. **MongoDB не требуется**: бэкенд работает на локальном файле `backend/data/db.json`.

## 2) Локальная “БД” (без Mongo)

При первом запуске сервер автоматически создаст `backend/data/db.json` и заполнит демо-данными (маршруты, точки, награды).

## 3) Установка и запуск Backend

Откройте терминал в папке проекта `ryazan-quest/backend` и выполните:

```bash
npm install
```

Запуск сервера (dev):

```bash
npm run dev
```

Команда `npm run seed` сейчас тоже просто инициализирует локальный `db.json` (по желанию).

## 4) Установка и запуск Frontend

Откройте терминал в `ryazan-quest/frontend` и выполните:

```bash
npm install
```

Запуск:

```bash
npm start
```

Открывайте:
- Frontend: http://localhost:3000
- Backend API: http://localhost:5000

## 5) Как тестировать функциональность

### Регистрация и вход

1. Откройте страницу `http://localhost:3000/register`.
2. Зарегистрируйтесь.
3. Войдите на `http://localhost:3000/login`.

### Карта и выбор маршрута

1. На главной странице выберите маршрут в блоке «Маршруты».
2. Карта покажет точки маршрута и линию пути.
3. Точки, которые уже сканированы, будут отмечаться зелёным цветом.

### Сканирование QR-кодов (онлайн)

1. Включите интернет.
2. Наведите камеру на QR-код.
3. Если QR совпадает с одним из `qrCodeValue` в seed, вы получите баллы.

Какие значения использовать (из seed):
- `ryazan_kremlin_01`
- `sobornaya_ul_02`
- `museum_youth_03`
- `esenin_statue_04`
- `gagarin_park_05`

Сгенерировать QR можно любым онлайн-генератором:
вставьте одно из значений выше, получите QR-код, затем отсканируйте.

### Офлайн-режим (сканы без сети)

1. Заранее подготовьте/загрузите данные онлайн:
   - зайдите на главную страницу и подождите, пока появятся маршруты/точки.
2. Затем выключите интернет (или включите режим “в самолёте”).
3. Отсканируйте QR-код: вы увидите сообщение, что сканирование сохранено офлайн.
4. Верните интернет обратно.
5. Приложение автоматически синхронизирует очередь (по событию `online`) и начислит баллы.

Как проверить, что офлайн действительно работает:
- при отключенном интернете маршрут/точки должны отображаться (берутся из IndexedDB),
- сканы должны ставиться в очередь и не падать из-за отсутствия сети.

### AI-гид

1. На карте нажмите «Подробнее» у точки.
2. Откроется окно AI-гид с фактами из `GET /api/points/:id/ask`.

> Примечание: сейчас AI-гид работает как интерактивный “факт-ассистент” на базе сохранённых ответов в БД.

### Лидерборд и магазин партнёров

1. «Лидерборд» запрашивает `GET /api/users/leaderboard` (обычно требует сети).
2. «Магазин партнёров» покупает награды через `POST /api/rewards/purchase` и списывает баллы.

## 6) Если что-то не запускается

1. Проверьте, что backend запускается без ошибок (должно быть создание `backend/data/db.json`).
2. В терминале backend ожидайте лог:
   - `Server running on port 5000`
3. Убедитесь, что в браузере нет CORS/сетевых ошибок.
